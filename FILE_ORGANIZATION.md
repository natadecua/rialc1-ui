# File Organization Guide for La Mesa Ecopark UI

This document provides an overview of the file organization structure in the La Mesa Ecopark LiDAR Tree Species Identification UI project. It explains what each directory and key file does.

## Directory Structure

```
rialc1-ui/
├── public/                  # Frontend files served to the browser
├── lamesa_forest_final_fixed/ # Map tiles for the application
├── raw_data/                # Source data files
├── Potree_1.8.2/            # Potree library for 3D visualization
├── node_modules/            # Node.js dependencies (generated by npm)
└── server.js                # Node.js Express server
```

## Key Files and Directories

### Server Files

- `server.js`: The Express server that serves the application. It handles:
  - Serving static files from the `public` directory
  - Serving map tiles from `lamesa_forest_final_fixed`
  - Serving raw data from `raw_data`
  - Serving Potree files from `Potree_1.8.2`
  
- `package.json`: Defines the project's npm dependencies and scripts.

### Frontend Files (`public/`)

- `index.html`: The main 2D map application HTML file. This is loaded when visiting the root URL.

- `script.js`: The JavaScript code that powers the 2D map interface, including:
  - Leaflet map initialization
  - Tree data loading and processing
  - Species and prediction visualization
  - Filtering and search functionality
  
- `style.css`: CSS styles for the 2D map interface.

- `lamesa_3d_viewer.html`: An embedded 3D viewer that shows LiDAR point cloud data with a side panel interface.

- `lamesa_potree_viewer.html`: A full-screen direct 3D Potree viewer for LiDAR point cloud visualization.

### Map Tiles (`lamesa_forest_final_fixed/`)

This directory contains pre-generated map tiles used by the Leaflet map. The tiles follow a TMS-compatible structure organized by zoom level, X, and Y coordinates.

See `TILE_MANAGEMENT.md` for details on the tile generation process and management.

### Data Files (`raw_data/`)

- `crown_shp/`: Contains shapefiles representing tree crown polygons.
- `shapefiles/`: Additional shapefiles related to the La Mesa Ecopark.
- `lamesa_processed.las`: The LiDAR point cloud data in LAS format.
- `lamesa_processed.copc.laz`: A cloud-optimized point cloud version of the LiDAR data.
- `prediction_results.csv`: Results from the machine learning model with tree classifications.
- Other supporting data files like GeoTIFFs and metadata.

### Potree Library (`Potree_1.8.2/`)

The Potree library used for 3D point cloud visualization. This includes:

- `build/`: Compiled Potree JavaScript files
- `libs/`: Supporting libraries needed by Potree
- `pointclouds/`: Directory where Potree looks for point cloud data
- Various example files and documentation

## Documentation Files

- `README.md`: Main project documentation
- `TILE_MANAGEMENT.md`: Guide for tile generation and management
- `3D_VIEWER_SETUP.md`: Documentation for setting up the 3D viewer

## Cleanup Scripts

- `cleanup_unused_files.ps1`: PowerShell script to remove obsolete test and development files
- `cleanup_unused_tiles.ps1`: PowerShell script to remove unused tile directories

## Development Workflow

1. The Express server (`server.js`) serves the application when you run `npm start`
2. When a user visits `http://localhost:3000`, they see `public/index.html` with the 2D map interface
3. The map loads tiles from `lamesa_forest_final_fixed` and shapefile data from `raw_data`
4. When the user clicks on 3D viewer links, they see the Potree-based 3D visualization
